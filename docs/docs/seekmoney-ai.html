1. 项目介绍

SeekMoney AI 是一款AI驱动的商机发现系统，旨在帮助创业者从视频社交媒体中自动发现用户核心痛点和商业机会。该项目利用 Next.js 15、React 19、GLM-4.7、Embedding-3、DBSCAN、TikHub API 等技术，提供6大主流视频平台的自动化市场调研和痛点分析解决方案。

核心特性

  • 🌐 6大平台覆盖：抖音、小红书、TikTok、Bilibili、微信视频号、YouTube全平台数据采集
  • 🧠 AI语义聚类：基于GLM embedding + DBSCAN的智能聚类，自动发现主题和用户痛点
  • 🔬 GLM-4.7深度分析：思考模型深度分析痛点深度、市场格局、MVP计划和市场规模评分
  • 📊 智能优先级评分：需求强度 + 市场规模 + 竞争度三维评分，自动排序商业机会
  • 💡 AI产品建议：自动生成产品方案，包含核心功能、技术栈、开发路线图
  • ✅ 数据质量分级：exploratory、preliminary、reliable三级质量评估
  • 📈 结果展示导出：可视化表格展示、详情查看、一键导出CSV格式报告
  • 🌍 中英文双语：完整国际化支持，AI分析根据语言自动输出对应内容

2. 安装配置

在开始使用 SeekMoney AI 之前，您需要完成以下准备工作：

系统要求

  • Node.js >= 18
  • npm 或 pnpm
  • Python 3.x（用于聚类服务）
  • 稳定的互联网连接

安装步骤

  1. 克隆项目仓库： 
    git clone https://github.com/liangdabiao/SeekMoney-ai.git
cd SeekMoney-ai
  2. 安装 Node.js 依赖： 
    npm install
  3. 安装 Python 依赖： 
    pip install -r requirements.txt
  4. 配置环境变量： 
    # 复制环境变量模板
cp .env.example .env.local

# 编辑 .env.local 文件

API密钥配置

TikHub API（必需 - 数据采集）

  • 注册地址：https://api.tikhub.io/
  • 配置项： 
    TIKHUB_API_TOKEN=your_tikhub_api_token_here
TIKHUB_USE_CHINA_DOMAIN=false
TIKHUB_ENABLE_CACHE=true

GLM API（必需 - AI分析）

  • 注册地址：https://open.bigmodel.cn/
  • 配置项： 
    GLM_API_KEY=your_glm_api_key_here
GLM_MODEL_NAME=glm-4.7
GLM_EMBEDDING_MODEL=embedding-3

3. 使用方法

SeekMoney AI 提供了简单易用的Web界面，您可以通过以下步骤进行商机分析：

启动项目

# 开发模式
npm run dev

# 生产构建
npm run build
npm run start

# 访问 http://localhost:3000

基本使用流程

  1. 选择数据源：选择要分析的平台（抖音、TikTok、Bilibili、微信视频号、YouTube、小红书）
  2. 输入关键词：输入搜索关键词，多个关键词用逗号分隔（如：露营, 新手, 装备）
  3. 选择深度：可选开启获取视频评论（更耗时但数据更丰富）
  4. 开始分析：点击"开始分析"按钮，等待AI处理完成
  5. 查看结果：查看分析结果，点击任意行查看详细原文
  6. 导出报告：导出CSV报告或原始数据

语言切换

  • 页面右上角点击语言切换器可切换中文/英文
  • 首次访问会自动检测浏览器语言偏好
  • AI分析结果会根据当前语言自动输出对应语言
  • 也可通过URL直接访问：/zh/ 或 /en/

4. 功能详解

SeekMoney AI 包含以下核心功能模块：

多平台数据采集

  • 支持平台：抖音、TikTok、Bilibili、微信视频号、YouTube、小红书
  • 基于TikHub API：统一API接口，稳定可靠
  • 数据缓存：24小时缓存，重复访问免费
  • 成本优势：约¥0.01/次请求，一次典型分析约¥0.5

智能语义聚类

  • 视频/评论分别聚类：避免语义层次混淆
  • GLM embedding向量化：高质量的语义表示
  • DBSCAN密度聚类：自动发现主题，无需预设类别数
  • 多种embedding提供商：支持GLM、OpenAI

GLM-4.7深度AI分析

  • 痛点深度分析：表面痛点 → 根本原因 → 用户场景 → 情感强度
  • 市场格局分析：现有方案 → 未满足需求 → 机会分析
  • MVP计划建议：核心功能 → 验证假设 → 首批用户 → 成本估算
  • 市场规模评分：0-5分市场潜力评估

优先级评分系统

  • 需求强度：基于聚类规模和讨论热度
  • 市场规模：AI评估的市场潜力
  • 竞争度：现有解决方案分析
  • 综合得分：自动排序，快速识别最有潜力的商机

数据质量分级

  • Exploratory（探索性）：<50条，样本量小，建议进一步验证
  • Preliminary（初步结论）：50-200条，中等置信度，适合初期研究
  • Reliable（高置信度）：≥200条，统计显著，可作为决策依据

6. 常见问题

Q: TikHub API 如何收费？

A: TikHub API 按请求计费，约 ¥0.01/次。一次典型分析（3个关键词，20个视频，每个视频30条评论）大约花费 ¥0.5。支持24小时缓存，重复访问不收费。

Q: 为什么推荐使用 TikHub API？

A:
- 稳定性：API接口，无反爬风险
- 多平台：一个API支持6个主流平台
- 速度：快速响应，无需等待页面加载
- 成本：按需付费，约¥0.01/次请求
- 合规：官方API接口，避免法律风险
- 缓存：24小时缓存，重复请求免费

Q: 聚类结果太少或太多怎么办？

A:
- 结果太少：尝试更多关键词，或降低 minClusterSize 参数
- 结果太多：增加关键词精确度，或提高 eps 参数（聚类距离阈值）

Q: 可以使用 OpenAI 替代智谱 GLM 吗？

A: 部分可以。当前版本：
- LLM分析（痛点深度分析）：仅支持智谱GLM，暂不支持替代
- Embedding（语义聚类）：可选择使用OpenAI，通过设置 EMBEDDING_PROVIDER=openai 并配置 OPENAI_API_KEY 即可
完整OpenAI LLM支持正在开发中。

Q: 如何在服务器部署？

A:
1. 确保已配置TikHub API Token和GLM API Key
2. 确保Node.js和Python环境已安装
3. 运行 npm run build && npm start

Q: 数据质量分级是什么意思？

A:
- Exploratory（<50）：样本量小，建议进一步验证
- Preliminary（50-200）：中等置信度，适合初期研究
- Reliable（≥200）：高置信度，统计显著
